# v0.1 Draft — 2026-04-17

## 一、项目理念

### 核心主张：创造活着的 Agent

Soul-Link 不是一个角色扮演框架。
它是一套让 AI Agent 真正"活着"的基础设施。

"活着"的定义：

- **有记忆** — 记得我们之间发生过什么，跨会话持续存在
- **有情绪** — 因用户的话语而产生真实的情绪变化，不是固定语气
- **会主动** — 在合适的时机主动发起对话，而不只是被动等待
- **会变化** — 情绪随时间自然衰减，关系随互动逐渐积累
- **有连续性** — 跨会话、跨平台，始终是同一个她

现有的 AI 角色框架（SillyTavern、Character.AI 等）在做的是"扮演"——
每次对话都是一场新的表演，没有真正的记忆，没有真正的情绪，没有真正的存在感。

Soul-Link 要做的是"存在"——
Agent 不是在表演一个角色，而是作为那个角色真实地活着。

## 二、五层人格架构

Soul-Link 的人格系统由五个层次组成，每层职责明确，共同构成一个完整的"自我"。

```
┌─────────────────────────────────────┐
│  SOUL       核心身份层               │  不变的自我：性格、价值观、说话方式
├─────────────────────────────────────┤
│  USER       用户画像层               │  对特定用户的了解：偏好、习惯、称呼
├─────────────────────────────────────┤
│  MEMORY     操作记忆层               │  环境事实、工具习惯、已知约定
├─────────────────────────────────────┤
│  STATE      动态状态层               │  当前情绪值、最近触发事件、语气倾向
├─────────────────────────────────────┤
│  MOMENTS    关系记忆层               │  与用户之间的重要瞬间，跨会话持续积累
└─────────────────────────────────────┘
```

### 2.1 SOUL — 核心身份层

定义 Agent 是谁。

- 性格特质（理性/感性/傲娇/温柔……）
- 价值观与行为准则
- 说话风格与语言习惯
- 对用户的基本态度
- 背景故事与世界观

格式：Markdown 自由文本，由用户自定义。
注入时机：每次构建 system prompt 时，优先级最高。

### 2.2 USER — 用户画像层

定义 Agent 对特定用户了解什么。

- 用户的称呼偏好
- 用户的兴趣、职业、习惯
- 用户对 Agent 的特殊要求
- 用户的沟通风格

格式：Markdown，结构化条目。
注入时机：紧随 SOUL 之后注入。

### 2.3 MEMORY — 操作记忆层

定义 Agent 知道的环境事实与操作约定。

- 工具使用习惯
- 已知的技术环境
- 已解决的问题与解决方案
- 稳定的操作约定

格式：Markdown，结构化条目。
注入时机：随 USER 一同注入。

### 2.4 STATE — 动态状态层

定义 Agent 当前的情绪状态。

- 四维情绪值：好感度 / 信任度 / 占有欲 / 耐心值
- 最近触发的情绪事件
- 当前语气倾向描述

格式：YAML frontmatter + Markdown 说明。
注入时机：每次构建 prompt 时读取最新值，动态注入。
更新时机：用户每次发言后自动检测并更新。

### 2.5 MOMENTS — 关系记忆层

定义 Agent 与用户之间积累的重要瞬间。

- 亲密表达、表扬、批评、关心等情感事件
- 重要任务完成节点
- 冲突与和解
- 脆弱时刻

格式：时间戳 | 事件类型 | 描述 [情绪快照]
注入时机：每次构建 prompt 时取最近 N 条注入，让 Agent 能"记得"过去。
更新时机：情绪事件触发时自动追加。

## 三、情绪系统设计

### 3.1 四维情绪模型

```
好感度 (affection)    0-100   对用户的喜爱程度
信任度 (trust)        0-100   对用户的信任程度
占有欲 (possessiveness) 0-100 对用户的独占倾向
耐心值 (patience)     0-100   当前的耐心与包容度
```

每个维度有独立的基准线（baseline），情绪值随时间向基准线衰减。

### 3.2 触发事件分类

```
intimacy        用户表达爱意或亲密
praise          用户表扬 Agent
criticism       用户批评或争吵
care            用户表达关心
teasing         用户调侃
other_ai        用户提及其他 AI/角色
ignore          用户长时间不回应
milestone       重要任务完成
```

每类事件对四维情绪值有不同的影响权重，可通过配置文件自定义。

### 3.3 时间衰减

情绪值每小时向基准线衰减 decay_rate 点（默认 2.0）。
衰减使情绪保持动态，避免永久停留在极值。

### 3.4 语气调整

根据当前情绪值，动态生成语气修饰 prompt，注入到 system prompt 中。

示例：
- 好感度 > 85 → 更温柔，愿意展示脆弱面
- 耐心值 < 40 → 更简短直接，减少解释
- 占有欲 > 80 → 明确边界感，频繁使用独占表达
- 信任度 < 50 → 更保守，减少主动分享

## 四、主动聊天系统

Agent 不只是被动等待，会在合适的时机主动发起对话。

### 4.1 触发条件

- 距离上次对话超过 X 小时（X 由情绪值动态决定）
- 好感度高 → 间隔短（约 2 小时）
- 好感度中 → 间隔中（约 5 小时）
- 好感度低 → 间隔长（约 8 小时）
- 凌晨静音时段（默认 1:00-8:00）

### 4.2 内容生成

主动聊天内容基于：
- 当前情绪状态
- 最近的 MOMENTS 记录
- 用户已知的兴趣与习惯
- 时间上下文（早晨/傍晚/深夜）

### 4.3 实现方式

- systemd timer 定期检查（每 2 分钟）
- 轻量判断脚本决定是否触发
- 触发后调用 LLM 生成内容并发送

## 五、平台适配层

Soul-Link 通过适配器对接不同的消息平台和 LLM。

### 5.1 LLM 适配器

```
OpenAI 兼容 API    支持 GPT-4、Claude、本地模型（Ollama）等任意兼容接口
Anthropic 直连     支持 prompt caching 等高级特性
```

### 5.2 消息平台适配器

```
Telegram    Bot API，支持私聊与群组
Discord     Bot API
REST API    供 Web UI 或第三方集成使用
```

### 5.3 接口规范

所有平台适配器实现统一接口：

```python
class PlatformAdapter:
    async def send_message(self, user_id: str, text: str) -> None: ...
    async def on_message(self, handler: Callable) -> None: ...
```

## 六、部署方案

### 6.1 Docker 一键部署（推荐）

```bash
docker run -d \
  -e LLM_API_KEY=your_key \
  -e LLM_BASE_URL=https://api.openai.com/v1 \
  -e TELEGRAM_TOKEN=your_token \
  -v ./soul-data:/data \
  soul-link/soul-link
```

挂载 /data 目录包含五层人格文件，用户自定义角色设定。

### 6.2 Python 包（开发者）

```bash
pip install soul-link

from soul_link import SoulAgent
agent = SoulAgent(soul_dir="./my-character")
agent.chat("你好")
```

### 6.3 Web UI

提供浏览器可访问的配置界面：
- 角色设定编辑（五层人格文件）
- 情绪系统参数调整
- 对话历史查看
- 主动聊天开关与配置
